# Minimal Product Example

This example shows what a small but realistic document-first workspace looks like after bootstrapping and first-pass planning.

Scenario
- Product: async voice-note request inbox for personal consulting / coaching / advisory workflows
- Goal: let visitors submit lightweight requests, let the owner triage them, and convert approved requests into scheduled sessions

Use this example to understand:
- what the baseline docs should look like
- how lane boundaries are expressed
- how task instructions connect to actual documents
